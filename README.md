# 40s-AI-Study

> **경력 10년+ 시니어 개발자를 위한 AI 실무 적용 스터디**
> 이론보다 실전, 트렌드보다 프로덕션.

---

## 🎯 우리는 왜 모였나

AI는 더 이상 선택이 아닙니다. 하지만 시중의 AI 콘텐츠는 대부분 **입문자용**입니다.

우리는 다릅니다.

- ✅ 이미 **레거시 시스템을 운영**해본 경험이 있습니다.
- ✅ **장애와 트래픽**을 겪으며 시스템을 키워본 경험이 있습니다.
- ✅ **팀을 리드**하거나 **아키텍처를 설계**한 경험이 있습니다.

이 경험 위에 AI를 얹을 때, 진짜 가치가 나옵니다.

**"Hello World" 말고, 내 회사 시스템에 어떻게 녹일 것인가"** 를 고민하는 스터디입니다.

---

## 👥 대상

- **10년차 이상** 백엔드/풀스택/데이터/인프라 개발자
- AI를 **실무에 적용**하고 싶은 분
- 단순 API 호출을 넘어 **아키텍처 레벨**에서 고민하고 싶은 분
- 서로의 **도메인 지식**을 나누며 배우고 싶은 분

> 📌 나이 기준은 느슨합니다. **시니어 관점의 실무 논의**가 핵심입니다.

## 📚 스터디 주제

### Core Topics

| 영역 | 내용 |
|------|------|
| **LLM 활용** | Prompt Engineering, Context 관리, Token 최적화 |
| **RAG** | 벡터 DB, 청킹 전략, 하이브리드 검색, 성능 튜닝 |

### 실무 케이스 스터디

- 사내 문서 검색 시스템에 RAG 적용하기
- 기존 API에 LLM 레이어 얹기
- CI/CD 파이프라인에 AI Agent 통합
- 로그/모니터링 시스템에 이상 탐지 붙이기

## 🛠 운영 방식

### 정기 세션
- **주기:** 격주 1회 (온라인/오프라인 병행)
- **형식:** 발표 30분 + 토론 30분 + 코드 리뷰 30분
- **기록:** 모든 세션은 이 Organization Repository에 아카이빙

### 참여 원칙
1. **Show, Don't Tell** — 코드와 실측 데이터로 이야기합니다.
2. **Production-Ready** — 장난감 예제가 아닌 실제 쓸 수 있는 수준을 지향합니다.
3. **Fail Fast, Share Faster** — 실패 사례를 적극적으로 공유합니다.
4. **Respect Experience** — 서로의 도메인 경험을 존중합니다.
